Password Breach Check
Uses the HaveIBeenPwned k-anonymity API. Your password never leaves your device — only a partial SHA-1 hash prefix is sent.
✓ SHA-1 partial hash only    ✓ Never stored    ✓ k-anonymity model
⚠ PASSWORD COMPROMISED

✓ PASSWORD NOT FOUND IN BREACHES
This password was not found in known breach databases. Still make sure it's long, unique, and not reused across services.
How It Works
1
Your password is hashed locally with SHA-1 in the browser — it never leaves your device as plain text.
2
Only the first 5 characters of the hash are sent to the HIBP API — this is called k-anonymity.
3
The API returns all hashes starting with those 5 chars. Your browser checks locally if your hash is in that list.
4
HIBP never sees your full password or hash — complete privacy preserved.
hash = SHA1("mypassword")
prefix = hash[0:5] // sent to API
suffix = hash[5:] // stays local
